# SECURITY POSTURE – Top-TieR Global HUB AI

## الهدف  
تأمين النظام بالكامل، من المتغيرات البيئية إلى الاتصالات الخارجية.

## المبادئ الأساسية  
- Zero-Trust Architecture  
- عدم تخزين أي بيانات حساسة في المستودع  
- جميع المفاتيح يجب أن تبقى في .env أو Secrets Manager  
- عدم إرسال أي API Key إلى سجلّات النظام

## طبقات الحماية  
### 1) حماية المتغيرات  
- جميع المفاتيح مخزنة في .env  
- تحميل آمن باستخدام `set -o allexport`

### 2) حماية الاتصالات  
- HTTPS إلزامي عند استدعاء النماذج الخارجية  
- استخدام مفاتيح محدودة الصلاحية (Least Privilege)

### 3) حماية الخدمات الداخلية  
- Redis و Neo4j يعملان على منافذ داخلية فقط (غير مكشوفة للإنترنت).  
- Telegram Bot يعمل عبر API وليس Webhook لمنع الهجمات الخارجية.

### 4) تهديدات متوقعة  
- سرقة المفاتيح  
- هجمات التلاعب بالاستجابات  
- هجمات DOS لو تم نشره كمخدم عام  

### 5) مسار الاستجابة للأعطال  
- جمع السجلات  
- تحديد المكوّن المعطّل  
- تشغيل health check  
- إعادة التشغيل التلقائي (Railway restart)
