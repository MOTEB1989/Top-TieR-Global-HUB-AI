[قالب طلب للمساعد عند مواجهة مشكلة أو رغبة في تطوير]

1) نوع الطلب (اختر أو عدّل):
   - إصلاح Workflow (GitHub Actions).
   - تحسين Docker أو Kubernetes.
   - تحسين أداء خدمة أو API.
   - إضافة ميزة جديدة.
   - مراجعة أمان (Security Review).
   - إعادة هيكلة كود (Refactoring).

2) الملفات/المقتطفات التي سأرسلها للمساعد:
   - لمسألة CI/CD:
     - محتوى ملف/ملفات:
       - .github/workflows/<اسم الملف>.yml
     - ملخص عن الخطأ، مثلاً:
       - صورة من GitHub للفشل.
       - أو نص رسالة الخطأ الأساسية (Job failed بسبب ...).

   - لمسألة كود Python/JS:
     - الموديولات/الملفات المعنية (أسماء الملفات + مقتطفات).
     - وصف مختصر لما يجب أن تفعله الدالة/الخدمة.

   - لمسألة Docker/Kubernetes:
     - محتوى Dockerfile إن وُجد.
     - محتوى docker-compose.yml إن وُجد.
     - محتوى ملفات k8s (مثل deployment.yaml, service.yaml).

3) الهدف النهائي:
   - ماذا تريد بالضبط أن يتحقق بعد التعديل؟
   - مثال: "أريد الـ Workflow ينجح ويفحص اتصال API الخارجية مع Logging واضح عند الفشل."

بعد تجهيز هذه النقاط، أرسل المحتوى للمساعد ليقدّم:
- تشخيص واضح.
- كود/ملف معدّل جاهز للاستبدال.
- شرح مختصر لما تغيّر ولماذا.
